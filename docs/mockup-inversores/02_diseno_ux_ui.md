## 02 Diseño UX/UI

- **Objetivo**: Definir la arquitectura de información, la experiencia de usuario y la estética visual del mockup.

### Actividades clave

- **Arquitectura de navegación**: Crear sitemap y definir flujo maestro de pantallas.
- **Wireframes de media fidelidad**: Representar estructura y jerarquías de contenido sin distracciones visuales.
- **Interacción y microcopys**: Detallar estados principales (éxito, error, vacíos) y textos que refuercen la narrativa de inversión.
- **Guía de estilos rápida**: Seleccionar tipografías, paleta, componentes base y patrones de gráficos/metas.
- **Prototipo navegable en Figma**: Conectar pantallas críticas para pruebas internas preliminares.

### Entregables

- Wireframes aprobados.
- Librería inicial de componentes UI.
- Prototipo Figma con flujo principal enlazado.

### Riesgos y mitigación

- Feedback disperso → calendarizar sesiones de revisión con checklists por disciplina.
- Falta de consistencia visual → documentar tokens y decisiones clave desde el día uno.
- Iteraciones lentas → utilizar componentes auto-layout y variantes reutilizables en Figma.
